HTTP REST Key-Value Store

Small HTTP REST key-value store built with Sinatra and Memcached.

Usage

Create or update

PUT /namespace/key
  • value is given as a parameter.
  • Update does not guarantee that key exists, because Memcached does not support atomic get+set operation. Also, because create or update operations update the full value of the specified key, a PUT operation is required over POST. See the excellent article "PUT or POST: The REST of the Story" by John Calcote.
  • $ curl -X PUT -d "value=looooong" kvs.api.enemy.fi/cats/longcat

Read

GET /namespace/key
  • If key is not found, then a HTTP 404 is returned.
  • $ curl kvs.api.enemy.fi/cats/longcat

Delete

DELETE /namespace/key
  • If key is not found, then a HTTP 404 is returned.
  • $ curl -X DELETE kvs.api.enemy.fi/cats/longcat
